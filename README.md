### 로또 구입 금액을 입력받는다

**정상적인 경우**

-[x] 올바른 숫자의 형태를 작성하는경우
    - ex) “1000”, “12000”
-[x] 1000원으로 나누어 떨어지는 경우
    - ex) “3000”, “18000”
-[x] 숫자의 범위는 0 ~ INTEGER.MAX_VALUE 범위여야 한다

**예외상황**

-[x] 숫자가 아닌 다른문자를 입력받은 경우
    - ex) “1,000”, “1_000”, “1!300”
-[x] 1000원으로 나누어 떨어지지 않는 경우
    - ex) “1300”, “2500”, ”3500”
-[x] 숫자의 범위를 벗어나는 경우
    - ex) 음수, 0, -1000, INTEGER.MAX_VALUE + 1
-[x] 로또 구입 금액을 통해 살 수 있는 로또의 개수를 반환한다.
-[x] 0으로 시작하는 숫자가 존재한다면 입력받을때 지우고 시작한다

**로또**

-[x] 번호는 무조건 6개여야 한다
-[x] 숫자는 1-45사이의 숫자를 가져야한다
-[x] 모두 다른 숫자를 가져야 한다.
-[x] 올바른 로또의 숫자를 받은 뒤 정렬해서 갖고 있어야 한다
-[x] 살 수 있는 로또의 개수만큼 로또를 구매한다.

### 당첨 번호를 입력 받는다. 번호는 쉼표를 기준으로 구분한다

**정상적인 경우**

-[x] 숫자와 쉼표만 입력받은 경우
-[x] 입력한 당첨 번호의 숫자 범위는 1~45 사이인 경우
-[x] 당첨 번호가 6개 오는 경우

**예외상황**

-[x] 숫자와 쉼표를 제외한 다른 문자를 입력받은 경우
-[x] 입력한 당첨번호의 숫자 범위가 1~45 사이가 아닌경우
-[x] 당첨번호가 6개가 아닌 경우

### 보너스 번호를 입력받는다.

**정상적인 경우**

-[x] 올바른 숫자형식인 경우
-[x] 숫자만 입력받은 경우
-[x] 보너스 번호의 숫자 범위가 1~45사이인 경우
-[x] 보너스 번호가 로또 번호를 제외한 다른 번호인 경우

**예외 상황**

-[x] 올바른 숫자형식이 아닌 경우
-[x] 숫자만 입력하지 않은 경우
-[x] 보너스 번호의 숫자 범위가 1~45 사이가 아닌 경우
-[x] 보너스 번호가 로또 번호와 같은 경우

### 당첨 통계 만들기
- [x] 입력받은 로또번호와 내가 갖고있는 로또 번호를 비교한다 
- [x] 맞은게 있는지 비교한다 
- [x] 수익률 계산해서 한자리수 까지만 찍기

### 공통 사항
- [x] 모든 입력은 잘못된 입력을 받았을시 다시 입력 받을 수 있다
